# Alt-Opinion Directive: Cross-Consistency Check — Specs ↔ Tasks

## intent:

Проверь связанность и целостность: спецификация infra-base → 4 задачи → Portal. Найди расхождения, пропуски, противоречия.

**Что проверить:**

1. **Покрытие:** Покрывают ли 4 задачи ВСЕ Bootstrap Requirements (17 строк) и ВСЕ 13 setup tasks из Handoff спецификации? Нет ли пропущенных конфигурационных файлов или пакетов?

2. **DAG корректность:** Правилен ли порядок зависимостей?
   - TSK-01 → TSK-02, TSK-01 → TSK-03, TSK-02+TSK-03 → TSK-04
   - Не создаёт ли это гонок (TSK-02 и TSK-03 параллельны, оба зависят от TSK-01 — всё ли в TSK-01 достаточно для их независимой работы)?

3. **Спецификация → Задачи:**
   - Все ли Verification Commands из спецификации (typecheck, svelte-check, test, test-e2e, lint, format, check, ci-check) имеют соответствующие проверки в тикетах?
   - Все ли Effective Rules правильно активированы в соответствующих фазах?
   - Нет ли задач, которые требуют tool/package, не упомянутого в Bootstrap Requirements?

4. **Фазовая структура:**
   - Корректны ли фазы внутри каждого тикета? Не пропущены ли необходимые шаги?
   - Правильно ли определены Rules для каждой фазы (config фазы без coding rules, test фазы с vitest-rules/playwright-e2e)?

5. **BDD покрытие:**
   - Достаточно ли Acceptance Criteria для инфраструктурного скопа?
   - Покрыты ли деградационные сценарии (npm registry down, Firebase недоступен, порты заняты)?
   - Есть ли тесты на конфигурационные файлы (валидность JSON/YAML/TS)?

6. **Execution Log:** Корректно ли заполнены PLAN-заглушки? Нет ли пропущенных фаз?

7. **Portal согласованность:** Соответствует ли `tasks/README.md` Tracker Index тому, что в `specs/README.md` Scopes table? Совпадает ли количество задач?

8. **Критические неудобные вопросы:**
   - Что произойдёт, если TSK-01 выполнен, а TSK-02 упал — остаётся ли проект в консистентном состоянии?
   - Как проверяется, что `npm run check` действительно вызывает ВСЕ заявленные инструменты?
   - Не создают ли параллельные TSK-02 и TSK-03 конфликт при модификации `vite.config.ts` (Vitest-конфиг может быть там же)?

**Формат ответа:**
- Критические расхождения (блокеры выполнения)
- Существенные замечания
- Рекомендации
- Итоговый вердикт: готово к выполнению / требует доработок
